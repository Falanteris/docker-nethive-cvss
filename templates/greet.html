Welcome to the automatic CVSS sumarrizer version {{ version }}

Redirect yourself to the /calculate/ url The API takes 3 parameters in based on sequence: /calculate/{vul}/{host and path to resource}

  • {vul} : The type of vulnerability. The current tag that works is SQLi and XSS at the moment

  • {host and path to resource} : The host (ip) that attacks the victim, the victim here is assumed to be the host that runs the CVSS API

Still confused?, you can get the feel of it by these following templates

Side Note: Calculation can take at most 15 seconds.. the low latency calculation is available when you run this in PIPELINE mode. But this would need you to have both Kafka and ElasticSearch, and the output would reach ElasticSearch first.

Usage Examples
/calculate/SQLi/rk-sragen.site/

Context: this would calculate an SQLi attack from rk-sragen.site on the local host's root path.

/calculate/XSS/rk-sragen.site/

Context: this would calculate an XSS attack from rk-sragen.site on local host's root path.

/calculate/XSS/rk-sragen.site/

Context: this would calculate an XSS attack from rk-sragen.site on local host's /vulnerable/ path. This would be unreachable by default, which should alter the final result.
